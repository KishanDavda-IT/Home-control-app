# Security Policy

## Supported Versions

We provide security updates for the following versions of the project:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

To report a security vulnerability, please use the [GitHub Security Advisory process](https://github.com/yourusername/lightfan-controller/security/advisories/new) or email security@lightfan-controller.example.

Please include the following information in your report:
- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker could exploit the vulnerability

This information will help us triage your report more quickly.

## Preferred Languages

We prefer all communications to be in English.

## Response Timeline

We aim to acknowledge receipt of your report within 48 hours and provide an initial assessment within 7 days. We will keep you informed of the progress towards a fix and full announcement, and may ask for additional information or guidance.

## Disclosure Policy

When we are ready to disclose a vulnerability, we will:
- Notify the reporter of our planned disclosure date
- Provide a detailed description of the vulnerability
- Explain the steps we have taken to mitigate the issue
- Share any relevant references (CVEs, etc.)
- Provide instructions on how users can protect themselves

We follow responsible disclosure practices and will work with reporters to ensure vulnerabilities are addressed appropriately before public disclosure.
